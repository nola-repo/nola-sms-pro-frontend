# Backend Handoff: Fixing the 500 Error (Slow Loading) in Sender Requests API

## The Issue
The frontend's "Active Sender IDs" section relies on `/api/sender-requests.php`. Currently, this endpoint throws a **500 Internal Server error** after hanging for about 10 seconds. 

This causes the "slow loading" the user experiences, ending with the list failing to populate on the UI.

The root cause is the following Firestore query in `api/sender-requests.php`:
```php
$query = $db->collection('sender_id_requests')
            ->where('location_id', '==', $locId)
            ->orderBy('created_at', 'DESC')
            ->limit(50);
```
Firestore **strictly requires a composite index** when combining a `where()` filter on one field and `orderBy()` on a different field. Because the index doesn't exist, the Google Cloud PHP SDK pauses execution and ultimately crashes with the error:
> `FAILED_PRECONDITION: The query requires an index.`

---

## The Solution
You have two options to fix this endpoint:

### Option A: Create the Composite Index in Firebase (Recommended)
You can simply follow the direct error link generated by the SDK to create the index with one click. 
Check your PHP error logs for the exact link, or manually go to the Firebase Console -> Firestore -> Indexes -> Composite and create:
* **Collection ID:** `sender_id_requests`
* **Fields:** `location_id` (Ascending) + `created_at` (Descending)
* **Query scopes:** Collection

### Option B: Sort in PHP Memory (No Index Required)
If you do not want to create an index in Firebase, remove the `orderBy` from the Firestore query and sort the results array manually in PHP before returning it.

**Update `api/sender-requests.php` (around line 30):**
```php
    if ($method === 'GET') {
        // Fetch sender ID requests without orderBy to avoid composite index requirement
        $query = $db->collection('sender_id_requests')
                    ->where('location_id', '==', $locId)
                    ->limit(50);
        
        $results = [];
        foreach ($query->documents() as $doc) {
            if ($doc->exists()) {
                $d = $doc->data();
                $results[] = [
                    'id' => $doc->id(),
                    'location_id' => $d['location_id'] ?? $locId,
                    'requested_id' => $d['requested_id'] ?? '',
                    'status' => $d['status'] ?? 'pending',
                    'purpose' => $d['purpose'] ?? '',
                    'sample_message' => $d['sample_message'] ?? '',
                    'created_at' => (isset($d['created_at']) && $d['created_at'] instanceof \Google\Cloud\Core\Timestamp) 
                        ? $d['created_at']->get()->format('Y-m-d H:i:s') 
                        : null
                ];
            }
        }

        // Sort descending by created_at in PHP memory
        usort($results, function($a, $b) {
            if (!$a['created_at']) return 1;
            if (!$b['created_at']) return -1;
            return strtotime($b['created_at']) - strtotime($a['created_at']);
        });

        echo json_encode($results);
        exit;
    }
```
